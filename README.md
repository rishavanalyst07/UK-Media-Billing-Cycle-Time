# UK-Media-Billing-Cycle-Time

Description - The project aims to analyze how long it takes for billing to happen after a media campaign runs, and identify delays or inefficiencies in the process.

1. Project Overview / Problem Statement

“This project was called the UK Media Billing Cycle Time Dashboard. The objective was to analyze the end-to-end billing process and identify delays occurring between media campaign execution, invoice receipt, and invoice booking. The business wanted visibility into billing cycle inefficiencies because delays were impacting cash flow tracking, invoice processing, and operational efficiency.”

2. Your Role and Contribution

“I worked as a Business Intelligence Analyst on this project. My responsibilities included extracting and validating billing data using SQL, transforming and cleaning the data in Power Query, creating KPIs and DAX measures in Power BI, and building interactive dashboards for Finance and Media teams. I also worked on delay bucket analysis, vendor-level performance tracking, and dashboard enhancements based on stakeholder requirements.”

3. Data Sources and Tools

“The data was primarily sourced from SQL Server databases containing media billing and invoice transaction data. SQL was used for data extraction and validation, Power Query for ETL and data transformation, and Power BI for dashboard development and visualization. DAX was used for KPI calculations and dynamic measures. Business users also utilized table-level exports for detailed invoice analysis.”

4. Process / Methodology

“First, billing and invoice data was extracted and validated from SQL databases. Then the data was cleaned, standardized, and transformed using Power Query. After that, I created data models and calculated cycle time KPIs using DAX formulas. The dashboard was designed with filters, delay buckets, vendor analysis, and detailed transaction-level visibility. Finally, UAT and data validation were performed with business teams before deployment.”

5. Key Features of the Solution

•Analyzed three key KPIs: Media Air Date → Client Bill Creation, Vendor Invoice Date → Invoice Receipt, and Invoice Receipt →
Invoice Booking to measure end-to-end billing efficiency.
•Designed delay bucket analysis (0–30, 31–60, 61–90, 90+ days) to identify billing bottlenecks, overdue invoices, and process gaps.
•Built vendor-level performance visuals to highlight high-delay vendors impacting overall billing cycle time.
•Delivered transaction-level visibility for Finance and Media teams to support invoice follow-ups, cash-flow monitoring, and
payment prioritization.

6. Impact and Results

“The dashboard helped Finance and Media teams identify bottlenecks in the billing process and track delayed invoices more efficiently. It improved visibility into vendor performance, supported faster invoice follow-ups, and helped stakeholders monitor billing cycle KPIs in a centralized manner. This reduced manual reporting efforts and improved operational decision-making.”

7. Key Learnings

“Through this project, I gained strong experience in operational analytics, KPI tracking, and cycle time analysis using Power BI. I improved my skills in SQL querying, DAX calculations, and dashboard performance optimization. I also learned how to translate business process challenges into actionable reporting solutions for Finance stakeholders.”

8. Future Enhancements / Monitoring Support

“Post-deployment, I supported dashboard monitoring, data validation, and enhancement requests from business users. Future enhancements could include automated alerts for overdue invoices, predictive analysis for delayed billing trends, vendor scorecards, and integration with automated workflow tools like Power Automate for notification handling.”
